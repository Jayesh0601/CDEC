# Detailed Intro

---

I have been working in the Cloud and DevOps domain, where my primary focus has been automating software delivery, managing cloud infrastructure, and ensuring application reliability.

Starting with source code management, I use Git and GitHub to maintain version control, manage branches, perform code reviews through pull requests, and collaborate with development teams.

For CI/CD, I work with Jenkins where I create and maintain pipelines to automate the complete software delivery process. The pipeline typically includes code checkout, build, code quality analysis, Docker image creation, image push to a registry, and deployment to Kubernetes environments.

For build automation, I have worked with Maven to manage project dependencies, compile source code, execute unit tests, and generate deployable artifacts.

To maintain code quality and security standards, I use SonarQube for static code analysis. It helps identify bugs, code smells, security vulnerabilities, and maintain quality gates before deployment.

For containerization, I use Docker to package applications along with their dependencies, ensuring consistency across development, testing, and production environments.

For container orchestration, I work with Kubernetes where I manage Deployments, Services, Ingress, ConfigMaps, Secrets, and Autoscaling. I am also involved in troubleshooting pods, analyzing logs, performing rolling updates, and ensuring application availability within the cluster.

On the cloud side, I have hands-on experience with AWS. I use EC2 for hosting applications and services, VPC for network isolation and security, IAM for access management and role-based permissions, and S3 for object storage, backups, and artifact storage.

I have worked with RDS for managing relational databases and Route 53 for DNS management and routing traffic to applications. To ensure high availability and scalability, I use Application Load Balancers and Auto Scaling Groups to distribute traffic and automatically scale resources based on demand.

For monitoring and alerting, I use CloudWatch to collect metrics, monitor resource health, create dashboards, and trigger SNS notifications whenever thresholds are breached.

To automate infrastructure provisioning, I use Terraform. I create reusable modules and manage AWS resources through Infrastructure as Code, which helps maintain consistency across environments and reduces manual configuration efforts.

For observability, I work with Datadog where I monitor infrastructure, Kubernetes clusters, application performance, logs, and custom metrics. I also configure dashboards and alerts to proactively identify and resolve issues.

On the Linux side, I regularly perform user and group management, file permission management, process monitoring, package installation, cron job scheduling, log analysis, storage management, and network troubleshooting using commands such as top, ps, netstat, ss, curl, ping, and traceroute.

Overall, my experience has provided exposure to the complete DevOps lifecycle—from code commit and build automation to infrastructure provisioning, application deployment, monitoring, and production support.
